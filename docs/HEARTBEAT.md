# HEARTBEAT.md - 每日检查清单

**项目**: 政府公文智能写作助手  
**更新频率**: 每日 2-3 次  

---

## 📋 每日检查项

### 1. 项目状态检查
- [ ] 查看 KANBAN.md 看板，确认任务进度
- [ ] 检查是否有紧急任务需要处理
- [ ] 确认代码是否已同步到 GitHub

### 2. 开发任务检查
- [ ] 当前 Sprint 任务进度
- [ ] 是否有阻塞任务需要协助
- [ ] 是否有新任务需要添加到 Backlog

### 3. 技术状态检查
- [ ] API 文档是否正常 (localhost:8000/docs)
- [ ] 是否有技术债务需要处理
- [ ] 是否有 bug 需要修复

### 4. 用户反馈检查
- [ ] 查看 Rui 的最新消息
- [ ] 收集功能改进建议
- [ ] 记录新需求

---

## 🔄 周期性任务

### 每周一次 (周一)
- [ ] 更新看板状态
- [ ] 回顾上周完成的工作
- [ ] 规划本周重点任务
- [ ] 更新项目进度图表

### 每两周一次
- [ ] 代码审查
- [ ] 性能基准测试
- [ ] 更新依赖包
- [ ] 备份知识库数据

## 🚨 紧急响应

当收到以下情况时，立即响应：

1. **生产环境故障** - 优先级: 最高
2. **用户阻塞问题** - 优先级: 高
3. **安全漏洞** - 优先级: 高
4. **紧急需求变更** - 优先级: 中
